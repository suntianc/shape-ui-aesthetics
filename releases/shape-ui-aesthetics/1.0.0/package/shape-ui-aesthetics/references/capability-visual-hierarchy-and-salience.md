# Visual Hierarchy & Salience

**Stable ID:** `visual-hierarchy-and-salience`  
**Class:** Core  
**Primary domain:** Information, Attention & Typography

## Observable trigger

Route here when attention lands in the wrong place, emphasis no longer matches consequence, grouping implies false relationships, or simultaneous focal claims leave the interface without perceptual rest.

## Non-trigger and near miss

Do not route here when users notice and interpret the element but cannot physically acquire it, when decision overload comes from too many choices, or when the core problem is unreadable text rather than priority. A tasteful hierarchy that merely differs from stakeholder preference is not a trigger without task or thesis evidence.

## Decision question

What should be noticed first, next, and only on demand, and which perceptual relationships will make that order match task consequence?

## Interpretation method

Treat attention as limited and goal-conditioned. Build an explicit attention map before styling. Use proximity, similarity, continuity, closure, figure/ground, common movement, scale, value, color, space, shape, position, and motion as evidence of grouping and priority.

Strengthen a focus first by inhibiting competing facts: fuse, simplify, quiet, darken, reduce edge activity, or increase rest around secondary regions before making the focal object louder. Preserve essential background narrative and data. Use emphasis as a scarce resource and never rely on color or motion alone.

## Executable design procedure

1. List the user's current goal, highest-consequence action or state, supporting evidence, and deferrable detail.
2. Draw the intended first, second, and resting attention zones.
3. Audit actual grouping through proximity, similarity, continuity, closure, figure/ground, and common movement.
4. Identify every competing edge, contrast, label, badge, animation, or frame-proximity pull.
5. Inhibit nonessential competition before increasing focal contrast.
6. Assign distinct contrast channels to priority, warning, selection, and background roles.
7. Test with blur, grayscale/value-only inspection, brief exposure, keyboard focus, zoom, and relevant color-vision or reduced-motion conditions.
8. Restore any suppressed context required for comparison, narrative, safety, or recovery.

## Boundaries and failure modes

- Do not make every important item visually exceptional; sequence or stage simultaneous importance.
- Avoid ad-like treatment that invites banner blindness.
- Do not use color-only or motion-heavy emphasis.
- Do not suppress background facts that carry essential data, story, orientation, or evidence.
- Signal Spatial Composition when massing creates the priority error, Interaction when affordance or target acquisition fails, and Human Fit when expectation filters the element out.

## Attached Reference Knowledge

Perceived groups become assumed task groups. Visual scope can imply action scope. A focus may gain solidity and luminosity when nearby facts are quieted rather than when it is overworked. Salience depends on culture, habit, task, device, and assistive technology, so perceptual contrast must be verified in the actual context.

Ask whether a brief glance finds the right consequence, whether visible grouping matches task relationships, whether focus survives without color or motion, and whether quieting secondary regions removed information users still need.
