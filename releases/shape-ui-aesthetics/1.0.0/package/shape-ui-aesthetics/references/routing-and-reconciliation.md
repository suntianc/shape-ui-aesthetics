# Routing and Reconciliation

## Route from evidence

Start a Routing Cycle when a checkpoint, new constraint, artifact observation, or absorbed contribution exposes a consequential unresolved decision. Scan all nine domains lightly; do not search the capability library globally.

Record each Open Decision with:

- a stable decision key;
- observed trigger evidence;
- the consequence of leaving it unresolved;
- dependencies and whether they are satisfied;
- the confidence or validation obligation.

Cluster decisions only when they share an outcome or must be interpreted together. Put every dependency-ready cluster in the Decision Frontier. Inside one cluster, select one to three contracts whose responsibilities are complementary. This limit is local to a cluster, not global to a checkpoint or assignment.

Core capabilities are conditional, not automatic. A Specialist requires its exact narrow trigger. A topic word, possible relevance, or desire for comprehensiveness is not a trigger. Reference Knowledge never routes independently.

## Require compact contributions

Make each capability return one internal delta:

```text
decision: <decision key>
evidence: <observed trigger evidence>
recommendation: <specific governing relationship or action>
consequence: <what changes in the artifact or direction>
boundary: <hard limit, failure condition, or tradeoff>
validation: <confidence and what must be checked>
onward: resolved | reframe | validate | human-choice
```

Do not accept mini-reports. A capability may signal another domain, but it must not load or invoke another capability. Return the signal to the Director for the next scan.

## Classify relationships before resolving them

- **Materially identical** — keep one method and retain the strongest relevant evidence and boundary.
- **Complementary** — assign distinct responsibilities inside the same decision.
- **Hierarchical** — let the higher-level governing relationship constrain local expression.
- **Mutually exclusive** — use the Decision Authority Ladder; never average incompatible recommendations.

Shared vocabulary or adjacent subject matter does not prove duplication.

## Apply the Decision Authority Ladder

1. Let explicit user and product constraints, core-task truth, safety, accessibility, cultural permission, and applicable hard boundaries veto incompatible proposals.
2. Reconcile ordinary interpretation against the accepted protagonist, ritual, thesis, core task, design grammar, current artifact, and observed evidence.
3. Ask the accountable user to decide equally defensible questions of identity, brand character, cultural position, emotional meaning, or preference.

Evidence maturity changes confidence, not aesthetic authority. Numerical consensus does not decide a conflict. Verify consequential current facts; if a factual conflict remains unresolved, keep the affected judgment provisional or suspended.

## Converge

Absorb reconciled contributions into the shared direction state. Re-scan only when the absorption creates a new material conflict, changes a dependency, or reveals new evidence. Converge when the current frontier has been absorbed and no dependency-ready material decision remains.
