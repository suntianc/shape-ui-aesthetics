# Security policy

## Reporting a vulnerability

Do not open a public issue for a vulnerability involving arbitrary command execution, credential exposure, unsafe network access, malicious Skill instructions, or compromised distribution archives.

Instead, report it privately to the repository owner through the contact route on the repository profile. Include a minimal reproduction, affected version or commit, impact, and any safe mitigation you have confirmed. Please do not include secrets in the report.

## Supported releases

Security fixes are evaluated against the current accepted stable release. Immutable historical release artifacts are retained for verification; a correction is published as a new accepted version rather than modifying an existing archive.
