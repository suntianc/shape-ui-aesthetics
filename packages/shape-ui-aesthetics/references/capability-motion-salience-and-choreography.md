# Motion Salience & Choreography

**Stable ID:** `motion-salience-and-choreography`  
**Class:** Core  
**Primary domain:** Motion, Rhythm & Causality

## Observable trigger

Route here when concurrent movement competes for attention, high- and low-consequence events use similar motion weight, or sequencing and stagger lack semantic hierarchy.

## Non-trigger and near miss

Do not route here for broken origin/destination, representation choice, or device performance alone. Simultaneous motion is legitimate when users must compare simultaneous events and attention remains legible.

## Decision question

Which moving event deserves attention first, how strong should each signal be, and what temporal order preserves comparison, response, and performance?

## Interpretation method

Treat motion salience as scarce. Rank events by frequency, urgency, risk, reversibility, and task consequence. Allocate one primary moving signal at a time where practical; merge, stagger, quiet, or make other signals static. Use multistep choreography only when properties have causal order, and stagger only when elements have reading order. Measure attention and technical peak load together.

## Executable design procedure

1. Inventory concurrent and recurring movement with trigger and consequence.
2. Rank events and identify the one to three current attention targets.
3. Map motion weight, distance, contrast, rhythm, and persistence to severity and frequency.
4. Keep critical feedback immediate; sequence subordinate events only when their order carries meaning.
5. Remove ambient and decorative competitors; preserve simultaneous comparison where required.
6. Measure peak concurrency, total completion, input response, power, and target hardware.
7. Test reduced motion, repeated use, screen reader independence, and real task urgency.

## Boundaries and failure modes

- Do not hijack attention like an advertisement or let a mascot become the task.
- Do not delay key feedback for ornamental stagger.
- Do not use motion as the only risk or warning channel.
- Avoid turning every event into an accent and extending total task time.
- Revert choreography that weakens comparison or merely masks a performance bottleneck.

## Attached Reference Knowledge

Motion carries unusually high attention weight. Severity determines signal strength; hierarchy determines sequence. Stagger can distribute visual and computational load, but delay is meaningful only when it expresses order.
